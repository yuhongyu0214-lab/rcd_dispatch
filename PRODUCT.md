# Product

## Register

product

## Users

运营管理员、调度员和后续接入人员在后台处理订单、车辆、司机和预警。使用场景是日常调度值守，需要快速扫描订单池、查看地图态势、完成派单、改派、撤回和日志追踪。

## Product Purpose

人车单调度系统用于把外部订单、司机、车辆、门店和调度规则连接成一个可演示、可联调的运营后台。当前目标是先跑通订单导入、地图看板、调度员闭环和推荐派单，再逐步接入真实云数据库、地图 ETA、外部订单和 GPS 数据。

## Brand Personality

克制、专业、调度感。界面需要像运营控制台，而不是营销页或普通蓝白 SaaS。

## Anti-references

不要蓝白 SaaS 默认风格。不要米白和纯黑强反差。不要大面积卡片堆叠、营销式 hero、装饰渐变、过圆组件或无意义动效。

## Design Principles

1. 以 `demo_v12` 的设计变量和页面布局为正式前端基准。
2. 优先保持调度员工作流稳定：信息密度、固定区域、内部滚动和按钮位置比装饰更重要。
3. 前端展示中文业务状态，后端保留英文枚举和统一接口契约。
4. HTML demo 只作为 UI/UX 参考，正式实现必须落在 Next.js、TypeScript、Prisma 和统一 API 响应内。
5. 地图、订单、司机、车辆、预警和日志按模块逐步接真实 API，不把 mock 数据写死进正式页面。

## Accessibility & Inclusion

面向后台操作，至少保证清晰对比度、键盘可达的原生表单控件、稳定焦点状态、可读字号和减少动画依赖。状态不能只靠颜色表达，必须配中文标签。
