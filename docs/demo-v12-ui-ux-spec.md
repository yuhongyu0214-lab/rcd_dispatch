# demo_v12 UI/UX 规格与设计变量约束

版本：2026-06-20  
来源：`.superpowers/live-preview/demo_v12.html` 当前定型版  
用途：作为正式 Next.js 实现的 UI/UX 输入，不作为正式 worktree 代码交付物。

## 1. 定位

`demo_v12` 已基本定型为人车单调度系统的前端操作体验参考。正式实现时应将本文件作为视觉、布局、交互和组件行为约束，按执行蓝图拆分到对应阶段：

- `map-board`：地图看板、点位、地图内对象切换。
- `admin-workflow`：订单详情、派单、改派、撤回、日志入口。
- `dispatch-rule-v1`：推荐派单 Top N。
- `logging-observe`：日志查询与 traceId 展示。
- `integration-adapter`：订单、车辆、GPS mock 数据接入。

原则：HTML demo 定义“长什么样、怎么交互、接口要什么数据”，正式代码必须使用 Next.js 14、TypeScript、Tailwind CSS、shadcn/ui、Prisma、统一 API 响应和 Pino 日志。

## 2. 设计变量

### 2.1 色彩

主视觉采用灰蓝底、暖锈色主色、低饱和绿/黄/红状态色。禁止回退为常见蓝白 SaaS 风格。

| Token | 当前值 | 用途 |
|---|---:|---|
| `--bg` | `oklch(0.945 0.006 235)` | 页面背景 |
| `--panel` | `oklch(0.905 0.012 235)` | 左侧工作面板 |
| `--surface` | `oklch(0.985 0.003 235)` | 卡片、输入框、表格底 |
| `--map` | `oklch(0.875 0.018 230)` | 地图底色 |
| `--nav` | `oklch(0.235 0.032 240)` | 左侧深色导航栏 |
| `--accent` / `--primary` | `oklch(0.635 0.115 35)` | 主操作、选中态、订单点位 |
| `--success` | `oklch(0.590 0.115 160)` | 在线、可调度、司机点位 |
| `--warning` | `oklch(0.690 0.125 76)` | 待处理、订单点位 |
| `--danger` | `oklch(0.610 0.160 27)` | 预警、风险按钮 |
| `--info` | `oklch(0.620 0.090 220)` | 车辆点位 |

色彩规则：

- 主按钮使用 `--accent`，风险按钮使用 `--danger` 的低饱和背景。
- 离线、暂不可用、GPS 离线使用灰色状态，不参与调度。
- 地图四类对象必须有稳定颜色和图标区分：订单水滴、车辆小车、门店小屋、司机方向盘、预警三角。
- 选中对象图标放大，其余对象透明度降低，用于突出当前关注对象。

布局规则：

- `body` 不使用页面级滚动；超出区域通过模块内部滚动轨道查看。
- 左侧 KPI、对象 tab、搜索区高度固定，不能因内容变化重排。
- 地图看板下方订单详情和推荐派单 Top N 固定高度，内容超出使用内部滚动。
- 表格采用横向轨道承载宽字段，不压缩到破坏字段语义。
- Chrome 与 Edge 下应保持同一设计比例；缩放由外层 `app-scale` 处理。

| `--shadow-modal` | `0 18px 46px rgba(31, 35, 39, 0.18)` | 地图浮层 |

| `--font-mono` | 等宽字体栈 | API、traceId、数值短码 |

字体规则：

- 不使用视口宽度动态缩放字体。
- 表格、KPI、卡片使用 tabular numbers。
- 按钮文字不得溢出；长字段通过 `title`、省略和横向滚动解决。

## 3. 全局布局

页面分为三层：

1. 左侧深色主导航栏：模块入口。
2. 左侧工作面板：当前模块 KPI、对象切换、筛选、列表。
3. 右侧主工作区：地图看板或模块信息工作区。

主导航模块：

- 订单池
- 地图看板
- 司机管理
- 车辆管理
- 预警中心
- 日志查询

右侧工作区规则：

- 地图看板只在地图 step 展示地图。
- 其他 step 不展示地图，只展示对应业务信息。
- 地图看板内点击 `订单 / 司机 / 预警 / 车辆`，只切换地图内对象、KPI 和左侧列表，不跳转到其他 step。
- 非地图 step 点击左侧主导航才切换页面。

## 4. 模块规格

### 4.1 订单池

目标：展示外部订单自动传入后的待处理订单池。

列表规则：

- 左侧订单卡每张展示：订单类型图标、订单号、车牌、状态、用车时间段、门店、起终点、入库时间、traceId。
- 已完成订单不在订单池左侧卡片中展示。
- 每页默认 20 条，可选 50 / 100 条。
- 搜索与筛选选择即生效。

右侧订单明细：

- 每个订单占一行。
- 字段包含：订单号、类型、订单状态、车牌号、门店、当前司机、用车时间段、取车点、还车点、取车地址、还车地址、是否锁单、入库时间、订单来源、traceId、订单进展。
- 表格支持横向滚动。

按钮：

- `查看日志`：跳转到日志查询并按订单关联。
- `司机 / 派单`：打开或选中推荐司机，不提交真实派单。
- 地理编码坐标不在前端详情中展示，仅作为后端写库和地图输入。

### 4.2 地图看板

目标：集中展示订单、司机、车辆、门店、预警点位。

对象切换：

- `订单 / 司机 / 预警 / 车辆` 为地图内对象 tab。
- 选择对象后留在地图看板，KPI 与左侧列表切换为对应对象。
- 地图不做图层隐藏，只做当前对象高亮。

筛选：

- 时间筛选：全部 / 2 小时 / 4 小时 / 6 小时。
- 订单状态：全部 / 待取车 / 待还车。
- 搜索：支持订单、司机、车牌、门店等关键词。
- 所有筛选无需提交按钮，选择即生效。

地图点位：

- 订单：水滴状。
- 车辆：小车。
- 门店：小屋。
- 司机：方向盘。
- 预警：三角。

右上操作：

- 刷新点位按钮后续接 `GET /api/map`。
- 无异常时不重置视图。

底部 dock：

- 左侧订单详情卡固定高度。
- 右侧推荐派单 Top N 固定高度。
- 两侧内容超出走内部滚动。

### 4.3 司机管理

目标：查看司机状态、调度参与性和工单进度。

状态展示：

- `S1`：门店空闲。
- `S2`：返程空闲。
- `S3`：门店忙碌。
- `S4`：订单忙碌。
- `OFFLINE`：离线，灰色，不参与调度。
- `UNAVAILABLE`：暂不可用，灰色，不参与调度。

右侧工单进度轴：

- 每名司机一行。
- 横向展示：前序工单、当前负责、后续工单。
- 状态点向右推进，当前节点有轻量动画。
- 右侧展示预计完成时间。

### 4.4 车辆管理

目标：查看车辆位置、调度参与性、车辆维度经营指标。

左侧车辆卡：

- 展示车牌、车型、状态、所属门店、GPS 更新时间、当前订单、锁单、是否参与调度。
- 直接从车辆卡片开始，不展示“车辆信息自动传入”说明卡。

右侧车辆明细：

- 每辆车一行。
- 字段包含：车牌号、车型、当前所属门店、车辆状态、GPS 状态、GPS 更新时间、当前订单、本月完单、本月营收、当前位置、参与调度、锁单。
- 上方局部搜索框支持车牌号、车型、门店、GPS 状态模糊筛选。
- 输入即筛选，不需要确认按钮。
- 正式接后端时，文本输入建议 300-500ms debounce 后请求接口。

### 4.5 推荐派单

目标：在地图看板或订单详情中展示 Top N 候选司机。

展示字段：

- 排名。
- 司机姓名。
- 优先级。
- ETA。
- 负载惩罚。
- 状态 / outcome。

交互规则：

- 保留点击候选司机即选中。
- 不保留“确认派给此司机”“刷新推荐”按钮。
- 选择无需提交按钮，自动更新选中态。
- 中文推荐理由在 demo 中删除；正式 `dispatch-rule-v1` 可由接口返回并按产品决策展示。

### 4.6 预警中心

目标：展示需要运营介入的异常。

排序：

- 不按类型分组。
- 按超过阈值时长降序排列，超过越久越靠前。

预警类型：

- 未接单超时。
- ETA 超限。
- GPS 离线。
- 低油量。
- 地址解析失败。

操作：

- `定位点位`：跳转地图看板并选中相关对象。
- `转人工处理`：记录操作日志，不直接改变订单状态机。

### 4.7 日志查询

目标：按订单号、司机、车牌号、操作类型查看操作留痕。

字段：

- 动作。
- orderId。
- driverId。
- driverName。
- licensePlate。
- operator。
- traceId。
- timestamp。
- result。

日志入口：

- 订单卡、订单详情、预警卡均可进入日志。
- 日志页面必须是真实 tab/panel，不使用 toast 替代。

## 5. 交互规则

### 5.1 无提交按钮筛选

适用：

- 前端已加载数据量较小。
- 下拉筛选。
- 当前页内局部模糊搜索。

正式后端接入规则：

- 文本输入使用 300-500ms debounce。
- 下拉选择立即请求。
- 保留重置入口。
- 不保留“查询/提交”按钮，除非查询成本高或需要组合条件确认。

### 5.2 按钮分级

- 主按钮：派单、选中推荐司机。
- 次按钮：查看日志、查看路径、定位点位。
- 风险按钮：改派、撤回、转人工处理、取消订单。
- 所有风险按钮必须有日志记录或二次确认。

### 5.3 状态与空态

每个模块至少具备：

- 一个正常演示状态。
- 一个空态。
- 一个异常或待复核状态。

空态文案短句表达，不加入说明型大段文字。

## 6. 正式实现验收

- Chrome 与 Edge 中 100% / 125% 缩放下模块不漂移。
- 左侧 KPI、tab、筛选区、地图底部 dock 高度稳定。
- 地图看板切换对象不跳转 step。
- 所有筛选选择即生效。
- 车辆明细局部搜索不影响左侧列表。
- 操作按钮能回答：点击后展示什么、调用哪个 mock 函数、未来接哪个 API。
- HTML demo 不直接并入正式 worktree，仅作为规格参考。
