## 1. 订单状态总览

| 状态值（代码枚举） | 中文名 | 含义 |
|-------------------|--------|------|
| `UNIMPORTED` | 待导入 | 订单在外部平台存在，尚未进入系统 |
| `PENDING` | 待分配 | 已入库，等待调度员发起派单 |
| `RECOMMENDING` | 推荐中 | 系统已计算推荐结果，等待调度员确认 |
| `ASSIGNED` | 已派单 | 调度员已确认派单，等待司机接单 |
| `ACCEPTED` | 已接单 | 司机已接单，任务尚未开始执行 |
| `IN_PROGRESS` | 执行中 | 司机已开始执行任务 |
| `COMPLETED` | 已完成 | 司机已完成任务，订单关闭 |
| `RECYCLED` | 已回收 | 超时未接单或人工撤回后回到待分配池 |
| `CANCELLED` | 已取消 | 订单被取消，不再进入调度流程 |

## 3. 各状态详细说明

### 3.1 UNIMPORTED — 待导入

- **触发条件**：订单在外部平台（哈啰 / 三方 SaaS）存在，尚未通过导入流程进入系统
- **退出条件**：通过手动 Excel 导入或 API 自动导入，字段校验通过后入库，状态变为 `PENDING`
- **说明**：此状态不存储于系统数据库，仅作为导入前的逻辑描述

---

### 3.2 PENDING — 待分配

- **触发条件**：
  - 订单成功导入入库（初始状态）
  - 已派单订单被调度员撤回
  - 司机拒单或超时未接单，系统自动回收后重新进入
  - 执行中订单因异常（车辆不可用等）被人工回收
- **退出条件**：调度员在后台对该订单触发推荐或直接手动派单
- **允许操作**：发起推荐、手动派单、取消订单

---

### 3.3 RECOMMENDING — 推荐中

- **触发条件**：调度员点击"推荐派单"，系统调用 `runDispatch()` 计算 Top N 候选司机
- **退出条件**：
  - 调度员确认某位司机 → 状态变为 `ASSIGNED`
  - 调度员放弃推荐结果 → 状态回到 `PENDING`
- **允许操作**：确认派单、取消推荐（回到 PENDING）、取消订单
- **注意**：同一时刻一条订单只能有一个推荐中状态，防止重复触发

---

### 3.4 ASSIGNED — 已派单

- **触发条件**：调度员确认推荐结果或手动指定司机，系统写入 `assignments` 表，同时触发调度锁
- **退出条件**：
  - 司机接单 → `ACCEPTED`
  - 司机拒单 → `RECYCLED`
  - 超时未接单（超时阈值由运营配置）→ `RECYCLED`
  - 调度员主动撤回 → `RECYCLED`
  - 调度员改派（指定新司机）→ 先 `RECYCLED` 再重新 `ASSIGNED`
- **允许操作**：撤回、改派、取消订单

---

### 3.5 ACCEPTED — 已接单

- **触发条件**：司机在移动端点击"接单"，系统更新订单状态，司机状态变为 S4（订单忙碌）
- **退出条件**：
  - 司机点击"开始执行" → `IN_PROGRESS`
- **允许操作**：开始执行（司机端）、调度员可在此状态改派（特殊情况，需记录原因）

---

### 3.6 IN_PROGRESS — 执行中

- **触发条件**：司机点击"开始执行"
- **退出条件**：
  - 司机点击"完成任务" → `COMPLETED`
  - 出现车辆不可用等异常，调度员人工干预 → `RECYCLED`（重新分配）
- **允许操作**：完成任务（司机端）、调度员人工处理异常

---

### 3.7 COMPLETED — 已完成

- **触发条件**：司机点击"完成任务"，系统记录完成时间，更新司机状态为 S1（门店空闲）
- **退出条件**：终态，不再流转
- **允许操作**：查看历史记录、导出日志

---

### 3.8 RECYCLED — 已回收

- **触发条件**：
  - 司机拒单
  - 超时未接单（`ASSIGNED` 状态超时）
  - 调度员主动撤回（来自 `ASSIGNED`）
  - 执行中出现异常，人工回收（来自 `IN_PROGRESS`）
- **退出条件**：重新进入 `PENDING`（待分配），由调度员再次发起派单
- **说明**：`RECYCLED` 是中间过渡状态，正常情况下会在秒级内自动变为 `PENDING`。改派场景下，系统先将订单置为 `RECYCLED` 再立即重新派给新司机，历史 assignment 记录保留。

---

### 3.9 CANCELLED — 已取消

- **触发条件**：调度员在任意状态下取消订单（`COMPLETED` 除外）
- **退出条件**：终态，不再流转
- **说明**：取消操作需记录操作人和取消原因，写入 `operation_logs`

## 4. 司机状态联动

订单状态变化时，对应司机状态同步更新：

| 订单事件 | 司机状态变化 |
|----------|-------------|
| 调度员确认派单（`ASSIGNED`） | 司机被调度锁锁定，状态标记为待确认（S3 短暂过渡） |
| 司机接单（`ACCEPTED`） | 司机状态 → S4（订单忙碌） |
| 司机开始执行（`IN_PROGRESS`） | 司机状态保持 S4 |
| 司机完成任务（`COMPLETED`） | 司机状态 → S1（门店空闲） |
| 订单撤回 / 回收（`RECYCLED`） | 调度锁释放，司机状态恢复为派单前状态 |
| 司机拒单（`RECYCLED`） | 司机状态 → S1（门店空闲） |

---

## 5. 超时与自动回收规则

| 场景 | 超时条件 | 自动处理 |
|------|----------|----------|
| 已派单未接单 | `ASSIGNED` 状态超过配置阈值（V1 默认 30 分钟） | 自动变为 `RECYCLED` → `PENDING`，写回收日志 |
| ETA 超限 | 推荐引擎计算最优 ETA ≥ 120 分钟 | 不进入 `RECOMMENDING`，直接标记人工处理 |

## 6. 状态枚举（供 Prisma schema 参考）

```prisma
enum OrderStatus {
  UNIMPORTED    // 待导入（逻辑态，一般不存库）
  PENDING       // 待分配
  RECOMMENDING  // 推荐中
  ASSIGNED      // 已派单
  ACCEPTED      // 已接单
  IN_PROGRESS   // 执行中
  COMPLETED     // 已完成
  RECYCLED      // 已回收
  CANCELLED     // 已取消
}
```
