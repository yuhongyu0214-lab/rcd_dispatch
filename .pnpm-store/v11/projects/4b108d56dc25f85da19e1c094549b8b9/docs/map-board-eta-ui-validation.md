# 地图看板 ETA UI 演示验证

## 验证入口

- 页面：`http://127.0.0.1:3024/admin/map`
- 范围：前端展示验证，不写数据库，不触发真实派单确认。

## 验证链路

1. 打开地图看板
   - 预期：右侧顶部标题/API 三块不再展示，地图直接铺到顶部。
   - 预期：地图左上展示“导航预计到达”浮层。

2. 切换 ETA 模式
   - 点击“驾车 / 公交地铁”。
   - 预期：每个模式均展示主方案和备用方案。
   - 预期：主方案为最短到达时间，备用方案为次短到达时间。

3. 订单视角推荐司机
   - 点击左侧“订单”，再点击任一订单卡片。
   - 预期：右下标题为“推荐派单 Top N”。
   - 预期：候选项展示司机、目标订单、ETA、负载惩罚、状态、结果。

4. 司机视角推荐订单
   - 点击左侧“司机”，再点击任一司机卡片。
   - 预期：右下标题切换为“推荐订单 Top N”。
   - 预期：候选项展示订单、目标司机、ETA、负载惩罚、状态、结果。

5. 点位联动
   - 点击地图上的订单、司机、车辆、门店或预警点位。
   - 预期：底部详情卡片同步切换。
   - 预期：若选中司机，Top N 按司机视角推荐订单。

## 后续接入点

- ETA 浮层当前为前端 fallback 演示数据。
- 后续接入高德路径规划后，替换 `getEtaPlans()` 数据源。
- 推荐排序当前沿用前端展示 DTO，后续由 `runDispatch()` 和真实 ETA 结果驱动。
