# demo_v12 与 V1 执行蓝图对照评估报告

评估对象：`demo_v12.html`

参考文件：
- `skill/人车单项目 V1 Worktree 执行蓝图_final.md`
- `车队调度管理看板-设计规则.md`
- `demo_v12.html`

## 1. 总体结论

`demo_v12` 适合作为前端接入前的高保真演示原型：它已经覆盖地图看板、订单池、司机池、预警列表、推荐派单 Top N、点位联动、InfoWindow、toast 和 API 接入占位。

但按执行蓝图衡量，它不是某一个 worktree 阶段的交付物，而是把 `map-board`、`admin-workflow`、`dispatch-rule-v1`、`logging-observe`、`integration-adapter` 的部分表现混合到了一个静态 HTML。后续若要接入真实系统，需要拆回阶段边界，并补齐导入、派单闭环、日志、状态机和真实 API。

## 3. 未实现或缺口

### 3.1 repo-bootstrap / 登录

- 未实现登录页、登录态、管理员账号入口。
- 未体现 `/api/auth/signin` 或 `admin@dispatch.dev / admin123` 演示路径。

优化建议：
- 顶部或侧栏底部增加“调度员：运营管理员”身份区。
- 增加一个轻量登录态提示，不必在 demo 中做完整登录，但要让演示脚本 Step 1 有视觉落点。

### 3.2 order-import / 订单导入

- 未实现 Excel 上传入口。
- 未体现导入模板校验、逐行报错、地址转坐标、导入结果页、导入批次号。
- 当前订单是静态 mock 数据，不区分“已导入批次”。

优化建议：
- 在左侧导航或工作面板增加“导入订单”按钮。
- 增加“最近导入批次”小面板：批次号、成功数、失败数、导入时间。
- 增加导入异常按钮或状态：点击展示逐行错误示例。

### 3.3 map-board / 地图看板

- 当前是 canvas 仿地图，不是高德 JS API。
- 点位坐标是 0-1 归一化 mock，不是 `pickupLat/pickupLng` 或司机最新位置。
- 缺少“手动刷新点位”明确语义，`重置视图` 更像地图操作，不像数据刷新。

优化建议：
- 将 `重置视图` 拆为两个按钮：`刷新点位`、`重置视图`。
- 地图右下角增加数据来源说明：`订单坐标 / 司机最后上报位置 / 车辆 GPS mock`。
- InfoWindow 中补充订单号、状态、取车地址、还车地址，严格对齐 map-board 验收。

### 3.4 admin-workflow / 调度员闭环

- 当前仅支持“推荐后确认派单”，没有完整手动派单、改派、撤回。
- 没有订单详情页或详情抽屉。
- 没有改派原因输入，也没有历史 assignment 展示。
- 没有撤回后 `ASSIGNED -> RECYCLED -> PENDING` 的视觉表达。

优化建议：
- 在订单详情/InfoWindow 中按状态显示按钮：
  - `PENDING`：推荐派单、手动派单、取消订单
  - `ASSIGNED`：撤回、改派、查看司机
  - `ACCEPTED`：改派、查看任务
  - `COMPLETED/CANCELLED`：查看日志
- 增加“改派原因”内联抽屉，不用弹窗优先。
- 增加“操作日志”区域，显示派单、改派、撤回记录。

### 3.5 dispatch-rule-v1 / 推荐引擎

当前 `runDispatchPreview()` 是演示排序，和蓝图规则存在偏移：

- 未区分到店类和上门类订单的候选筛选规则。
- 未实现上门类“每个司机仅允许 1 个上门类订单”约束。
- 未实现同一门店维度筛选。
- 未实现门店空闲司机数量决定是否启用负载均衡。
- 未实现 `ETA >= 120` 返回 `MANUAL / ETA_EXCEEDED`。
- 未实现无司机返回 `PENDING / NO_DRIVER` 的显式 UI。
- 未体现高德 API 失败时 `etaMinutes = 9999` 并排末尾。
- `confirmAssign()` 直接修改本地状态，没有“事务防并发重复派单”的风险提示。

优化建议：
- 推荐面板增加 outcome 状态：`DISPATCHED / PENDING / MANUAL`。
- 推荐列表每一项显示三个解释维度：状态优先级、ETA、负载惩罚。
- 对 ETA 超限候选显示“人工处理”标识。
- 确认派单按钮文案改为 `确认派给此司机`，避免像“ETA 按钮”。

### 3.6 logging-observe / 日志与 traceId

- 页面只显示 toast，不显示 traceId。
- 没有响应头 `X-Trace-Id` 的演示占位。
- 没有导入、派单、改派、撤回的日志闭环。
- 左侧“日志查询”按钮只是 toast，占位太轻。

优化建议：
- 增加固定的“链路追踪”信息区，展示当前 `traceId`。
- 每次派单/撤回/改派后追加一条操作日志卡片。
- 日志查询按钮切换到日志 tab，而不是只弹 toast。

### 3.7 integration-adapter / 外部适配

- `API_ENDPOINTS` 是前端路径占位，但没有哈啰/GPS 适配层概念。
- 车辆管理入口仅 toast，没有车辆 GPS mock 数据。

优化建议：
- 增加“外部同步”按钮或状态条：哈啰订单 mock、GPS mock。
- 车辆 tab 至少展示车牌、车辆状态、GPS 更新时间。
- 在 UI 文案中区分“订单来源：Excel / 哈啰 mock / 三方 API mock”。

### 3.8 stabilization / 演示脚本

蓝图推荐演示脚本包含 8 步，当前 demo_v12 可覆盖 Step 3-6 的视觉演示，但缺 Step 1、2、7、8：

| 演示步骤 | demo_v12 支持情况 |
|---|---|
| 登录系统 | 未支持 |
| 导入订单 | 未支持 |
| 打开地图看板 | 支持 |
| 点击订单标记查看详情 | 支持，但字段不全 |
| 触发推荐派单 | 支持 |
| 查看 Top N 并确认派单 | 支持 |
| 对已派单订单执行改派 | 未支持 |
| 查看操作日志 | 未支持 |

## 4. 前端布局优化建议

### 4.1 信息架构

当前左侧为导航，中栏为列表，地图右侧为推荐面板，方向正确。建议补成四层工作流：

1. 左侧导航：地图、导入、订单、司机、车辆、日志、设置。
2. 中栏工作池：根据当前导航切换订单池、司机池、导入结果、操作日志。
3. 地图主画布：点位、路径、刷新、图层筛选。
4. 右下决策面板：订单详情、推荐结果、操作按钮。

### 4.2 按钮体系

建议统一按钮分级：

| 类型 | 按钮 | 位置 |
|---|---|---|
| 主操作 | `运行推荐`、`确认派单` | 订单详情或推荐面板 |
| 次操作 | `手动派单`、`改派`、`撤回` | 订单状态操作区 |
| 数据操作 | `刷新点位`、`同步哈啰订单`、`同步 GPS` | 地图顶部或工具条 |
| 导航操作 | `查看日志`、`查看导入结果` | 中栏 tab 或左侧导航 |
| 风险操作 | `取消订单`、`强制回收` | 详情区底部，需原因输入 |

### 4.3 状态驱动按钮可见性

按钮应按订单状态显示，避免所有动作同时暴露：

| 订单状态 | 建议按钮 |
|---|---|
| `PENDING` | 推荐派单、手动派单、取消订单 |
| `RECOMMENDING` | 确认派单、放弃推荐、刷新推荐 |
| `ASSIGNED` | 撤回、改派、查看司机 |
| `ACCEPTED` | 改派、查看任务 |
| `IN_PROGRESS` | 查看任务、异常回收 |
| `COMPLETED` | 查看日志 |
| `RECYCLED` | 重新进入待分配、查看回收原因 |
| `CANCELLED` | 查看日志 |

### 4.4 推荐面板优化

当前推荐面板持续显示，建议改为随订单上下文变化：

- 未选订单：显示“选择一条 PENDING 订单开始推荐”。
- 选中 PENDING：显示推荐按钮和候选列表。
- 选中 ASSIGNED：显示当前司机、撤回、改派。
- ETA 超限：显示 `MANUAL` 人工处理卡片。
- 无司机：显示 `NO_DRIVER`，给出“稍后重试 / 手动派单”按钮。

### 4.5 接入态提示

当前 `API mock connected` 太笼统。建议拆成：

- 地图数据：`/api/map`
- 推荐接口：`/api/dispatch/recommend`
- 确认派单：`/api/dispatch/confirm`
- 日志追踪：`traceId`

这样前端 demo 会更像“准备接入真实系统”的工具，而不是纯视觉稿。

- 增加车辆 tab 和 GPS mock。
- 将 `重置视图` 拆为 `刷新点位` 和 `重置视图`。

- 左侧导航增加文字 tooltip 或展开态。
- 推荐按钮不要常驻地图顶部，移到订单详情上下文。
